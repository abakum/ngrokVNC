Распутать запутанные параметры:

1) Cторона требующая помощи (Подчинённый) показывает экран (VNC server) - поэтому ngrokVNC запускается без параметров `ngrokVNC` - так им проще
1.1) Подчинённый запустил `ngrokVNC` первым. Если есть интернет - создатся туннель.
     Сторона оказывающая помощь (Главный) должна запустить `ngrokVNC :` (VNC viewer)
     параметр `:` это разделитель ip и порта. Так как туннель создан то ни ip ни порт не указываем это атрибуты туннеля
     Дальше можно не читать - это и есть основной режим использования ngrokVNC
     Так как оператор ngrok туннеля это ЧеловекПосередине то лучше использовать UltraVNC для шифрации трафика.
     
1.2) Подчинённый запустил `ngrokVNC` первым. Если нет интернета - нужно сообщить Главному свой ip. Главный должен запустить `ngrokVNC ip`
     Нет туннеля - можно не шифровать трафик и пользоваться TightVNC 
     
1.3) Главный запустил `ngrokVNC 0` первым. Создаётся туннель. Подчинённый должен запустить опять `ngrokVNC` - так им проще
     параметр 0 это порт 5500
 
2) Параметр `-` это реверс. Обычно Подчинённый ждёт подключения Главного чтоб показать экран, но можно и наоборот - `-` это наоборот
2.1) Главный запускает `ngrokVNC -0` первым когда нет интернета и говорит Подчинённому свой ip. Подчинённый должен запустить `ngrokVNC -ip`
2.2) Если есть интернет Подчинённый может запустить `ngrokVNC -` ngrokVNC будет ждать пока Главный не запустит `ngrokVNC 0` который создаст туннель

3) Если Подчинённых несколько и они за NAT - можно запустить один UltraVNC репитер и использовать один туннель для всех.
   Параметр `+` это добавка нескольких Подчинённых к тоннелю репитера

3.1) Запустим репитер в режиме I.

3.1.1) Подчинённый запустил `ngrokVNC` на хосте с ip и сообщил Главному свой ip
       Если есть интернет - создатся туннель не к Подчинённому а к репитеру
       Главный для помощи должен запустить `ngrokVNC :ip`
       параметр `:` как и в 1.1)  параметр `ip` как в 1.2)
3.1.2) Подчинённый2 запустил `ngrokVNC +` на хосте с ip2 и сообщил Главному свой ip2
       Главный для помощи должен запустить `ngrokVNC :ip2`

3.2) Запустим репитер в режиме II. 
3.2.1) Подчинённый запустил `ngrokVNC` на том же хосте где и репитер его `id` будет 0
       Если есть интернет - создатся туннель не к Подчинённому а к репитеру
       Главный для помощи должен запустить `ngrokVNC :` или `ngrokVNC :0`
       параметр `:` как и в 1.1) параметр `0` это адрес от 0 до 999999999
3.2.2) Подчинённый2 запустил `ngrokVNC +2` в том же LAN что и у репитера и сообщил Главному что его id `2`
       Главный для помощи должен запустить `ngrokVNC :2`
3.2.3 Подчинённый c IP 192.168.1.2 в том же LAN что и у репитера  после создания туннеля может запустить `ngrokVNC`
      это назначит ему ID:19216812
3.2.4) Главный может запустить `ngrokVNC :1` первым и собщить Подчинённому что его id `1`
       Подчинённый для получения помощи должен запустить `ngrokVNC +1`
